## 0. 작업 방식 (수기 모드 — 이 프로젝트의 규칙)

1. **코드는 커서 인덱스로 표기** — 모든 코드는 `줄ID: 코드` 형식으로 제시하고, 동근님이 에디터에 **직접 타이핑(수기)** 한다. 통짜 붙여넣기 금지.
   - 파일 접두사: `a`=app.py · `d`=dicom_io · `e`=deid · `p`=preprocess · `i`=infer · `r`=report · `s`=store
   - 수정 지시는 줄ID로 짚는다: "d4를 이렇게 바꿔"
2. **한 번에 한 파일, 작은 단위** — 파일 하나 완성·실행 확인 후 다음으로.
3. **스택은 Python + Streamlit만** — 별도 프론트엔드/백엔드 서버 없음.

## 1. 화면 구성 (Streamlit 페이지 4개)

**① 업로드** — DICOM 파일(병원 X-ray 표준 파일)을 끌어다 놓으면:
- 환자 이름·생년월일 같은 개인정보 태그를 자동으로 지우고
- 사진에 글자로 박힌 환자정보도 블러 처리
- 원본 vs 비식별본을 나란히 비교해서 보여준다

**② 분석** — [분석 시작] 버튼:
- 진행률 바(전처리 → AI 판정 → 보고서 초안)가 단계별로 차오르고
- 결과: **"이상 소견 의심 / 정상 범위" + 확신도 %** + 처리 시간

**③ 보관함** — 지금까지 분석한 목록(비식별 ID 기준):
- 클릭하면 상세 결과 + **보고서 초안**(LLM 생성, 품질 게이트 통과본)
- 모든 화면 하단 고정 문구: *"교육·기술 데모입니다. 진단이 아니며 의료적 판단은 의사에게."*

**④ 아레나 (6차원 · 50개 · 서바이벌 — 그대로 유지)** — 보고서 **작성자 페르소나(=프롬프트 구성) 50개**를 6차원(역할·연차·어조·구조·근거 표기·설명 난이도)으로 생성 → 벤치마크 케이스 채점 → 하드 게이트(진단 단정 즉시 해고)·합성 70점 컷·쌍대결로 **1~10위만 생존, 40개 해고** → **1등 구성이 ③의 운영 프롬프트로 채택**된다. 리더보드에 점수 분해와 해고 사유 표시.
- 경쟁 단위가 "가짜 의사 패널"이 아니라 "보고서 작성자"인 이유: 회사가 실제로 하는 일(프롬프트 최적화)과 동형이라 면접에서 통한다. 6차원·해고 규칙·임계값 설계는 원안 그대로.

## 2. 폴더 구조 — 화면은 얇게, 로직은 core/에

```
medgate/
├── app.py                  # Streamlit 화면 3페이지 (화면 코드만, 얇게)
├── core/                   # ★ 진짜 로직 — Streamlit 없이 동작·테스트 가능 (면접에서 보여줄 코드)
│   ├── dicom_io.py         # [신규] pydicom으로 DICOM 읽기 — 태그 추출, 환자/검사 ID 계층
│   ├── deid.py             # 비식별화 — 태그 제거 + 사진 속 글자 블러   ← plateguard app/core.py 이식
│   ├── preprocess.py       # X-ray 보정(CLAHE·감마 등)                ← yolo11 preprocessor.py 이식
│   ├── infer.py            # AI 판정 (근골격 X-ray 정상/이상 분류)
│   ├── report.py           # 보고서 초안 생성 + 품질 게이트(진단 단정 금지어 차단·재시도) ← n8n 채점 패턴 이식
│   └── store.py            # SQLite 저장·조회                         ← yolo11 db.py 이식
├── scripts/
│   ├── prep_data.py        # [신규] 샘플 DICOM + MURA(공개 근골격 X-ray) 준비
│   └── bench.py            # 처리 속도 실측                           ← plateguard scripts/_bench.py 이식
├── tests/                  # pytest — core/만 테스트 (회귀 게이트, yolo11 문화 이식)
├── data/                   # 샘플·결과 (대용량은 gitignore)
├── requirements.txt        # streamlit, pydicom, opencv-python, numpy, ultralytics(또는 torch), google-genai, pytest
├── .env.example            # GEMINI_API_KEY (보고서 초안용 — 호출 적음)
└── plan.md
```

**왜 이렇게 나누나**: Streamlit은 화면일 뿐이고, `core/`가 백엔드 코드다. 채용 공고가 "백엔드 개발자"라서, 면접관에게 보여줄 것은 app.py가 아니라 core/와 tests/다. 나중에 필요하면 FastAPI 파일 하나 얹어서 API 서버로 확장 (선택, D6).

## 4. AI 모델 (D2에 A/B 결정)

- **데이터**: pydicom 내장 샘플(개발·테스트용) + **MURA** — 스탠포드 공개 근골격 X-ray ~4만 장, 무료. 지원 회사 도메인(근골격)과 동일 부위
- **A안(기본)**: MURA 일부로 분류 모델 직접 파인튜닝 — "모델 직접 학습" 스토리를 의료에서 재현 (이미 번호판으로 해본 일)
- **B안(폴백)**: 공개 가중치로 파이프라인 먼저 완성. A안은 나중에 교체
- 보고서 초안: Gemini(보유 키), 게이트 심사는 temperature 0, 호출량 적어 비용 무시 가능
